# Руководство по AI Usage

Это руководство для новых пользователей. Оно помогает настроить `AI Usage` с нуля, чтобы устройство показывало использование Claude или Codex.

## 1. Перед началом

Сначала проверьте:

1. Устройство включено.
2. Устройство и компьютер находятся в одной локальной сети.
3. Вы можете открыть страницу `settings.html`.
4. Вы знаете IP-адрес устройства.
5. На компьютере установлен Python 3.

Проверка Python:

```bash
python3 --version
```

В Windows можно также использовать:

```bash
py --version
```

## 2. Как работает AI Usage

`AI Usage` показывает на устройстве локальные данные использования AI.

Поддерживаются:

1. Claude
2. Codex

Нужно запустить Python-скрипт на компьютере:

1. Скрипт читает локальные данные.
2. Скрипт отправляет данные на устройство по HTTP.
3. Устройство обновляет приложение `AI Usage`.

Claude и Codex используют один и тот же endpoint:

```text
/api/claude_usage
```

## 3. Настройка Claude

### Шаг 1

Откройте `AI Usage` на устройстве.

### Шаг 2

Скачайте:

```text
claude_usage.py
```

Прямая ссылка:

<https://raw.githubusercontent.com/GeekMagicClock/smalltv-pro-s3/main/tools/claude_usage.py>

### Шаг 3

Запустите:

```bash
python3 claude_usage.py 192.168.1.123
```

Замените `192.168.1.123` на реальный IP устройства.

Windows:

```bash
py claude_usage.py 192.168.1.123
```

### Шаг 4

Оставьте скрипт запущенным и держите устройство на странице `AI Usage`.

## 4. Настройка Codex

### Шаг 1

Откройте `AI Usage` на устройстве.

### Шаг 2

Скачайте:

```text
codex_usage.py
```

Прямая ссылка:

<https://raw.githubusercontent.com/GeekMagicClock/smalltv-pro-s3/main/tools/codex_usage.py>

### Шаг 3

Запустите:

```bash
python3 codex_usage.py 192.168.1.123
```

Windows:

```bash
py codex_usage.py 192.168.1.123
```

### Шаг 4

Проверьте, что локальные данные сессий Codex существуют.

Обычный путь:

```text
~/.codex/sessions
```

В Windows обычно:

```text
C:\Users\ВашеИмя\.codex\sessions
```

Если сессий еще нет, сначала используйте Codex один раз.

## 5. Частые проблемы

### На устройстве нет обновления

Проверьте:

1. Правильный IP
2. Одна и та же сеть
3. Устройство открыто на `AI Usage`
4. Скрипт все еще работает
5. Есть локальные данные Claude или Codex

### Ошибка сети

Обычно это значит:

1. Неверный IP
2. Устройство офлайн
3. Другая сеть
4. Изоляция клиентов на роутере
